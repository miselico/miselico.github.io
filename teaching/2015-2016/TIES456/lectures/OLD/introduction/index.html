Services

A service comprises a stand-alone unit of functionality available via a formally defined interface.

Services should follow the following principles

  • Abstraction
  • Autonomy
  • Composability
  • Discoverability
  • Formal contract
  • Loose coupling
  • Reusability
  • Statelessness

Why do you want to study services?

  • Services are ubiquitous and are the main building block for the Internet.
  • All companies’ IT infrastructure is based on services.
  • Service orientation makes adapting to change easier.
  • Services uses similar concepts as modular programming and interface based design.
  • Defining isolated services makes testing easier.
  • See later for more …

Cloud computing

  • Started as a business term
  • Cloud computing is the use of computing resources (hardware and software) that are delivered as a service over a network. (Wikipedia)
    • Thus services come first
  • The business aspects have difficult consequences on the implementation side
    • Security becomes a big issue when allowing multi tenancy

Why do you want to study cloud computing (1/2)?

  • To make your boss happy …

Why do you want to study cloud computing (2/2)?

  • More and more services are moved to the cloud
    • When developing services, you must know cloud computing
  • It becomes too expensive to manage hardware yourself - especially with low load.
    • But it becomes cheaper again at some point.

Sorting some int

  • In class exercise
    • Several implementations of the same functionality were analyzed
    • It deemed impossible to find a best implementation
  • A non-exhaustive list of factors which indicate ‘performance’
    • Speed
      • Processing cycles (how many steps)
      • Wall clock time (how much time when executed - in parallel )
    • Space optimization = memory consumption
    • Readability
    • Usability
      • Broadness of the domain
    • Development time
      • Familiarity with the language and tools
    • Portability
    • Maintainability
