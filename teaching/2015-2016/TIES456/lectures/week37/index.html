General notes - Streams or String

    EntityUtils.toString(...)
    int k;
    String st = "";
    while ((k = stream.read()) != -1) {
        st += (char) k;
    }
  • All create in memory stored strings.
    • Fine for small deployment
      • A disaster for large scale deployments (if the string is very large)
  • Try to use streams instead

General note - Streams and Readers

  • Reader/Writer
    • Used for characters (a .txt file, JSON, XML, …)
  • InputStream/OutputStream
    • User for bytes - binary data (e.g. an image, sound, …)
  • InputStreamReader and OutputStreamWriter bridge between the byte and chatacter world
    • You can and should specify a character encoding for the conversion
      • otherwise some default will be used (= bad things will happen to ö, ä, 错, …).
    • Why is there no ReaderInputStream and WriterOutputStream ?

General feedback - ErrorChecking

  • Only implemented well by very few groups
    • If the HTTP reponse code 200 = OK
      • The HTTP communication succeeded!
    • Check in the response message whether the call also succeeded.
      • "result":"Success" vs "result":"Fail"
